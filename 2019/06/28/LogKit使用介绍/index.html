LogKit使用介绍

发表于 2019-06-28 更新于 2020-03-23 分类于 经验总结

LogKit 是高通提供的一个日子抓取工具，一般可以通过 Launcher 直接进入或者通过拨号盘串号（可配置）进入，在 app 界面可以进行常规的 start/stop 、配置log内容等操作，功能强大。

但因为在游乐园二代相机上，无法通过屏幕进行交互，所以需要其他方法对 LogKit 进行控制，与高通人员沟通后，确认可以是通过广播的方式。

打开LogKit

adb shell am start -n com.qualcomm.qti.logkit/.cActivity

开启日志抓取

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Logging Start

停止日志抓取

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Logging Stop

修改配置

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e SwitchOn “Configs/Modem/Golden”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e SwitchOn “Configs/Modem/Audio”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e SwitchOff “Configs/Modem/Golden”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e SwitchOff “Configs/Modem/Audio”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Button “Settings/Android/Bug\ Report”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Button “Settings/Android/Surface\ Flinger”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Picker “Settings/Storage-internal”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Picker “Settings/Storage-sdcard1”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Picker “Settings/Storage-sdcard0”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Picker “Settings/Log\ chunk\ size-5”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Picker “Settings/Log\ chunk\ size-50”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Picker “Settings/Log\ chunk\ size-500”

打包日志

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation -e Package “Sessionname-sdcard0”

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation –esn Discard

获取LogKit状态

adb shell am broadcast -a com.qualcomm.qti.logkit.intent.action.cAutomation.Automation –esn GetState

需要注意的是，通过该广播获取状态的原理是：发送一个有序广播，附带一个Key为“GetState”，Value 为空字符串的 Extra 信息，LogKit 内部会有一个优先级较高的广播接收器进行拦截，然后通过 setResultExtras(Bundle bundle) 把状态加到该广播中，然后再继续转发。所以我们需要自己再监听 com.qualcomm.qti.logkit.intent.action.cAutomation.Automation 这个广播，然后通过 getResultExtras().getString(“State”) 来获取 LogKit 状态，值为“Running”和“Stopped”

预置配置

LogKit 的一些设置项虽然可以通过广播来处理，但对我们来说更有用的是在编包之前就提前预置好这些设置项，配置文件位置如下：

vendor/qcom/proprietary/prebuilt_HY11/target/product/msm8953_64/vendor/etc/qti-logkit/qti_logkit_config.xml

另外，如果相机版本是USERDEBUG，也可以通过 push 修改过的配置 xml 文件到 /system/vendor/etc/qti-logkit/ 目录下，然后重启设备来应用新的配置
