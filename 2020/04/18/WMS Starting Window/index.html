WMS Starting Window

发表于 2020-04-18 更新于 2020-04-21 分类于 学习笔记

概览

定义

Starting Window：TYPE 类型为 TYPE_APPLICATION_STARTING 的 Window，且每个 Activity 最多只能有一个 Starting Window

实际上就是在点开 app 之后的一个启动页面，默认是一个空白，但实际上三方 app 会在 Starting Window 上展示 logo 或者加入一些推广广告之类的，主要是为了给window加载一个缓冲时间

生命周期

从 AMS 启动/重启 一个 Activity 到 Activity 的任意一个窗口绘制完成

类型

  • STARTING_WINDOW_TYPE_NONE

  • STARTING_WINDOW_TYPE_SNAPSHOT

    • 使用应用退出时候的快照
    • 热启动

  • STARTING_WINDOW_TYPE_SPLASH_SCREEN

    • 使用 app theme 里配置的背景
    • 冷启动或内存较低

添加 Starting Window

入口：showStartingWindow

在启动 Activity 的过程中，会通过 ActivityRecord.showStartingWindow() 开始添加 Starting Window

ActivityRecord.addStartingWindow()

同样也是处理一些异常情况做 return，然后调用到 WindowAppToken.addStartingWindow()

我们来挨个看看这些方法的实现

findMainWindow

AppWindowToken.findMainWindow()

此方法优先寻找当前AppWindowToken下的所有（非子窗口）类型为TYPE_BASE_APPLICATION或者TYPE_APPLICATION_STARTING的窗口中，没有在执行退出动画（mAnimatingExit为false）的窗口；如果没有mAnimatingExit为false的窗口，则就可以返回mAnimatingExit为true的

getStartingWindowType

AppWindowToken.getStartingWindowType()

有三种情况会为启动的Activity添加starting窗口：

  1. 在一个新task中启动此Activity
  2. 此Activity所在的进程还未运行
  3. 此Activity的task和启动它的Activity不在同一个task并且此Activity还未创建

最常见的添加启动窗口的情况就是在Launcher上点击一个应用启动
