WMS Focus Window

发表于 2020-04-14 更新于 2020-04-17 分类于 学习笔记

定义

每次 add Window，remove Window，relayout Window 都会改变 Window 的显示次序，拥有焦点的 Focus Window 也会发生变化。

Focus Window 决定了是哪个窗口来接收 按键事件和屏幕点击滑动事件

流程

入口：updateFocusedWindowLocked()

WindowManagerService.updateFocusedWindowLocked()

RootWindowContainer.updateFocusedWindowLocked()

DisplayContent.updateFocusedWindowLocked()

DisplayContent.findFocusedWindowIfNeeded()

DisplayContent.mFindFocusedWindow

这个更新 Focus Window 的策略还是很好理解的。首先几种特殊情况：

  • token 被移除了，直接返回 false
  • 当前没有 Focus Window，则直接把当前遍历的窗口设置为 Focus Window
  • 当前的 Focus Window 不能被 Focus，也直接把当前遍历的窗口设置为 Focus Window

接下来就是通过 compareTo 把当前 Focus Window 和遍历的窗口进行比较，当前窗口优先级大于遍历窗口，则跳过此轮遍历，相反则设置遍历窗口为新的 Focus Window
