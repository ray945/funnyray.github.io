AWS IoT KeepAlive 心跳流程

发表于 2019-06-15 更新于 2020-03-23 分类于 经验总结

首先简单说一下，AWS Iot SDK 的 mqtt 模块是基于 paho 标准的 mqtt client java 进行开发的，没有加入 paho 的 android 扩展包，所以心跳只能通过 TimerPingSender 进行定时发送。但由于在 android 系统中，如果系统长时间没有 cpu 活动，会进入休眠状态，而简单的 Timer 是无法唤醒 cpu 的，这就导致官方的 AWS IoT SDK 无法理想地运行在游乐园项目（相机需一直保持 mqtt 在线）。所以游乐园项目使用的 AWS IoT SDK 是我们手动加入了android 扩展包中的 AlarmPingSender 后自己编译的。

提炼超类（Extract Superclass）

两个雷有相似特性。为这两个类建立一个超类，将相同特性移至超类。

动机

  • 重复代码改动麻烦。
  • 两个类以相同方式做类似的事情，或以不同的方式做类似的事情，对象提供了一种简化这种情况的机制，那就是继承。
  • 另一种选择是 Extract Class。两种方案是继承和委托的区别。如果两个类可以共享行为，也可以共享接口，那么继承是比较简单的做法。
