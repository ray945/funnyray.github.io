发表于 2015-09-15 更新于 2020-03-23 分类于 经验总结

分析

刚开始在网上查了相关资料，发现这个问题是在测试前，待测设备已经添加了账户，经测试，移除账户后，此项可以pass

可是由于高通之前添加了相关代码使用户可以选择把联系人添加到本机还是sim卡，这导致手机每次重启，都会默认添加两个账户：PHONE和SIM

于是我尝试在Setting-Account中屏蔽PHONE和SIM两个账户，使这两个账户不会出现在Setting-Account中（其他手机的Account界面都不会有这两个账户），经测试，这种方法依然不行，因为机器上仍然存在那两个账户

所以只能从CTS错误提示入手。

首先根据错误提示定位到 gen/android/app/admin/IDevicePolicyManager.java， 发现这是一个接口，于是到OpenGrok上进行搜索，发现一个继承自IDevicePolicyManager.Stub的DevicePolicyManagerService.java类，就是它了！

找到setDeviceOwner方法

在这里我们看到了一句熟悉的错误提示

那么问题就出在这个判断条件了，也就是allowedToSetDeviceOwnerOnDevice()方法，定位过去

可以看到，会在这个方法里通过AccountManager获取到系统里所有已添加的账户到一个Account数组，接下来取得这个数组的长度，如果不为0，即系统中已添加了账户，返回false到上一层，抛出异常”Trying to set device owner but device is already provisioned.”

这里就是根本原因了，可以想到，解决办法很简单，调用这个方法的时候，传入一个参数packageName，加入一个判断，如果遍历过程中发现某个account是localphone或者sim，且传入的packageName是CTS或者GTS的包名，则让这个account数组减一，从而在测试的时候屏蔽这两个账户。
